# Описание вклада участников

Проект реализован соло: **Соколова Юлия**.

## Порядок реализации

1. **Парсинг данных** (`scripts/parser.py`) — сбор ≥5000 статей Wikipedia через
   MediaWiki API (обход `list=allpages` + батчевые `prop=extracts`) с
   бэкоффом на rate-limit и промежуточным сохранением на диск.
2. **Индексация в Elasticsearch** (`scripts/indexer.py`) — создание индекса
   `articles` с маппингом полей `title`/`content`/`url`, загрузка статей.
3. **Базовый поисковый сервер** (`app/main.py`, эндпоинт `GET /search`) —
   `multi_match`-запрос с fuzziness и подсветкой найденных фрагментов.
4. **Суммаризация результатов** (`app/summarizer.py`) — интеграция с Claude
   Haiku и экстрактивный fallback-режим (частотный отбор предложений) на
   случай отсутствия API-ключа; промпт содержит анти-галлюцинационную
   инструкцию (отвечать только на основе результатов поиска).
5. **Веб-интерфейс первой версии** (`static/index.html`) — форма поиска,
   отображение результатов и суммаризации.
6. **Слой пользователей, ролей и авторизации** (`app/auth.py`, `app/models.py`,
   `app/db.py`, `app/schemas.py`) — SQLite + SQLAlchemy, хэширование паролей
   через bcrypt, выдача и проверка JWT.
7. **Разграничение доступа и аудит** (`app/routers/history.py`,
   `app/routers/saved.py`, `app/routers/admin.py`) — история поиска, избранное
   с проверкой роли (writer/owner), админ-эндпоинты с JOIN/GROUP BY/подзапросами
   для статистики (owner-only).
8. **Доработка UI под роли** (`static/index.html`) — экран логина/регистрации,
   условный показ кнопки «Сохранить» и вкладки «Админ» в зависимости от роли,
   XSS-фикс при подстановке URL в `href`.
9. **Багфиксы и наведение порядка** — чтение `ELASTICSEARCH_URL` из `.env`
   вместо хардкода, замена устаревшего параметра `body=` в вызовах ES-клиента,
   добавление `.gitignore`.
10. **Документация к защите** (`README.md`, `docs/ER-diagram.md`,
    `docs/report.md`) — инструкция по запуску, ER-диаграмма, отчёт по практике.
11. **End-to-end проверка** (`scripts/smoke_test.py`, `scripts/seed_owner.py`) —
    скрипт создания первого owner-пользователя и smoke-тест, прогоняющий полный
    сценарий (регистрация → поиск → права ролей → админка) с проверкой в консоли.
12. **Правки по итогам ревью** — расширение коллекции с ~30 до ≥5000 документов
    (`scripts/parser.py` переписан на обход `list=allpages` вместо фиксированного
    списка тем), анти-галлюцинационная инструкция в промпте суммаризации, явное
    указание в отчёте, что Elasticsearch использует BM25 по умолчанию (закрывает
    требование «алгоритм поиска» без дополнительной реализации).
